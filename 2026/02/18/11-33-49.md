**Thread**: [thread](11-33-00.thread.md#:~:text=2026/02/18/11-33-49)

This patch adds the definition of the TRY_HANDLER_DATA macro for
the AArch64 architecture, as well as makes modifications to the
exception handler responsible for the __try/__except blocks.

The first change fixes a bug where the existing exception context
record is reused as an unwind context record, which fails with a
crash on Windows on Arm (AArch64). The fix is to create a new
record instead, while leaving the original one intact.

The second change adds an additional condition for cases when the
handler is called again in the second phase of unwinding on certain
platforms such as Windows on Arm (AArch64). In this case, the value
ExceptionContinueSearch is simply returned without making any changes.

Tests fixed on AArch64:
winsup.api/ltp/access03.exe
