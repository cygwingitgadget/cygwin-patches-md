> No, #ifdef __cplusplus won't help. Example:
>
> foo.h:
> extern "C" {
> #include <sys/wait.h>
> }
>
>
> sys/wait.h:
> ...
> #ifdef __cplusplus
> ...
> inline int __wait_status_to_int(int __status) { .... }
> // This fails because both are interpreted as C-functions:
> inline int __wait_status_to_int(const union wait &__status) { .... }
> ...
> #endif
>
>
> Fix:
> #ifdef __cplusplus
> +extern "C++" {
> ...
> inline int __wait_status_to_int(int __status) { .... }
> inline int __wait_status_to_int(const union wait &__status) { .... }
> ...
> +}
> #endif
